About

Quantum-resistant file encryption on the Minima blockchain

🔒

AES-256-GCM

Military-grade encryption standard. A 256-bit key makes brute-force computationally infeasible.

⚡

W-OTS+

Quantum-resistant one-time signature. Secure against attacks from future quantum computers.

🔑

PBKDF2-SHA512

250,000 iterations for key hardening. Significantly slows down any brute-force attempt.

🌐

Zero-Knowledge

Only 32 random bytes are visible in the encrypted file. No metadata exposed.

5 Layers of Protection
1

File Encryption

AES-256-GCM turns file contents into unreadable data. Built-in integrity verification detects any tampering.

2

Key Protection

The encryption key is wrapped using PBKDF2-SHA512, bound to your node's seed phrase. Without the node, the key cannot be extracted.

3

Concealed Metadata

Address, signature, encryption parameters — all encrypted. An attacker cannot determine even how the file was protected.

4

Quantum-Resistant Signature

W-OTS+ guarantees the file was created by you and has not been modified. Resistant to quantum computer attacks.

5

Node Binding

The file can only be decrypted on a node with your seed phrase. Offline brute-force is physically impossible.

How it differs from standard encryption
Standard encryption: file + password → decrypt. An attacker can brute-force the password offline, without any limits.

Quantum Crypto: file + live node + seed phrase → decrypt. An offline attack is physically impossible — without the node, there is nothing to brute-force.
